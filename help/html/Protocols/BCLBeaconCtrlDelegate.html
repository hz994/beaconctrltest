– shouldAutomaticallyPerformAction: required method

Called to ask a delegate if a BeaconCtrl action should be handled automatically by the SDK

- (BOOL)shouldAutomaticallyPerformAction:(BCLAction *)action

Parameters

action

An action that would normally be handled by default by the SDK

Return Value

YES, if the action should be handled automatically

Discussion

Some BeaconCtrl actions, e.g. URL actions, are handled automatically by the SDK by default (in case of URL actions a modal web view is shown). You can use this method to prevent this default behavior.

– didChangeObservedBeacons: required method

Called each time a set of beacons currently monitored by the SDK has changed

- (void)didChangeObservedBeacons:(NSSet *)newObservedBeacons

Parameters

newObservedBeacons

A set of new monitored beacons.

Discussion

The SDK monitors a limited set of beacons at any given time and changes this set basing on the device’s position and beacons currently in range. This way, the SDK gets through the iOS limitation for 20 monitored beacons at a time.
